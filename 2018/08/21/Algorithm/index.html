Introduction

Move all zeros to the leftmost

Now we want to find an efficient algorithm to move all 0’s in a 0/1-array to the leftmost places which minimizes (say) the number of changes in the array-items.

use two pointers. One points to the rightmost zero and the other one points to the leftmost one and then swap them.

Path count

How to count #(acyclic-paths between two nodes in a complete digraph of N nodes) - contd.

  • (1$\to$ n with 1 step) : 1
  • (1$\to$ n with 2 steps) : n-2
  • (1$\to$ n with 3 steps) : (n-2)(n-3)
  • ….
  • (1$\to$ n with k steps) : (n-2)(n-3)…(n-k)
  • (1$\to$ n with (n-1) step) : (n-2)(n-3)…2*1

Dijkstra算法

求从顶点$v_1$到其他各个顶点的最短路径

  1. 设置dis数组

    数组中的元素值表征$v_1$到其他顶点的距离

    初始化顶点集为：$T=\{ v_1\}$, 路径是：$v_1$

    即$dis(v_1,v_1)=0$，$dis(v_1,v_2)=\infin$，$dis(v_1,v_3)=10$，$dis(v_1,v_4)=\infin$，$dis(v_1,v_5)=30$，$dis(v_1,v_6)=100$。

  2. 选择一个最短距离

    因为$dis(v_1,v_3)=10$是最短的，所以将$v_3$加入到顶点集中：$T=\{ v_1, v_3\}$

    路径是：$v_1 \to v_3$

    一旦有新的点加入，则我们更新dis数组，$v_3$可以到达的点有：$v_4$。

    $dis(v_1 \to v_3 \to v_4)=10+50=60< \infin $，所以更新$dis[3]=60$

  3. 继续选择一个最短距离

    除去顶点集$\{v_1,v_3\}$中的点，在dis中选择一个距离最小的点：$v_5$，加入顶点集$\{v_1,v_3, v_5\}$。

    路径是：$(v_1 \to t_3\to t_5)$

    更新dis数组，$v_5$可以到达的点是：$v_4$和$v_6$。

    $dis(v_1 \to t_3\to t_5\to t_4 )=30+20=50 < 60$，更新$dis(v_4)=50$

    $dis(v_1 \to t_3\to t_5\to t_6 )=30+60=90 < 100$，更新$dis(v_5)=90$

  4. 继续选择一个最短距离

    将$v_4$加入顶点集$\{v_1, v_3, v_5,v_4\}$，路径是：$(v_1 \to t_3\to t_5\to t_4)$

    $t_4$可以到达的点：$v_6$

    $dis(v_4 \to v_6)=50+10=60 < 90$，更新

  5. 选择一个最短距离

    将$v_6$加入顶点集$\{v_1, v_3, v_5,v_4, v_6\}$，路径是：$(v_1 \to t_3\to t_5\to t_4 \to v_6)$。

    $v_6$不能到达任何点。

  6. 结束

    因为$dis(v_2)=\infin $，则没有任何点可以到达它，所以，算法结束。

  • the link (x,y) is processed at most once.

    because x is closed at most once, (x,y) is processed when x is closed.

    d(y)=d(x)+c(x,y) if y is not open or closed.

    if d(y)>[d(x)+c(c,y)], d(y)=d(x)+c(x,y)

    everytime d(y) is changed, parent(y)=x

  • if G has N nodes, max number of links is N(N-1)=O(N^2), so processing all links takes at most O(N^2)

    the max number of acyclic paths from s to other nodes in G is $e(N-1)!$.

    When the graph is a complete digraph and because one link is processed at most once.

  • Time complexity for dijkstra’s algoritm is $O(N^2)$.

    Step 1 takes $O(N)$ time for initialization.

    Step 2 takes $O(1)$ time

    Step 3 and Step 4 has two loops:each loop takes $O(N)$ time, so two steps take $O(N^2)$

    SO the overall time complexity is $O(N)+O(1)+O(N^2)=O(N^2)$

    max number of closed nodes is O(N)

    computation to choose $i_{th}$ node for closing:

    $OPEN_1=\{s\}$

    $|OPEN|_2\le N-1$

    $|OPEN|_3\le N-2$

    ….

    $|OPEN|_{N-1}\le N-(N-2)=2$

    $|OPEN|_N\le 1$

    $OPEN_i=\{x_i,y_i,y_2,…,y_{N-i}\}$, because $|OPEN|_i$ is at most $N-(i-1)$, and in order to find a shortest path, we need to compare:

    $d(x_i)\le d(y_1)$

    $d(x_i)\le d(y_2)$

    $d(x_i)\le d(y_{N-i})$

    which requires at most $(N-i)$ comparisons

    so $OPEN_2$ requires at most $N-2$ comparisons in selecting $x_2$

    $OPEN_3$ requires at most $N-3$ comparisons in selecting $x_3$

    …

    $OPEN_{N-1}$ requires at most $1$ comparisons in selecting $x_{N-1}$

    SO THE total computation is $1+2+3+…+(N-2)=\frac{N^2-3N+2}{2}$

    • The initialization takes $O(N)$ time, where we make $d(x)=\infin$ for each node $x$.
    • The sum of all links weights computation is $O(N^2)$
    • The path of $s\to x_i$ is computed at most $i-1$, so computation of all paths is $(1+2+..+(N-1))=O(N^2)$.
    • Th total number of paths from starting node $s$ is $O(e\cdot (N-2)!)=O((N-2)!)$, ============================================================

  • Dijkstra Algorithm is not suitable for graph with negative path cost.
